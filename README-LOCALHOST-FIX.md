# 🔧 Исправление ошибки "Connection refused: localhost"

## Проблема

Если вы видите ошибку:
```
io.netty.channel.AbstractChannel$AnnotatedConnectException: 
Connection refused: getsockopt: localhost/[0:0:0:0:0:0:0:1]:5136
```

Это означает, что ваш компьютер пытается подключиться к `localhost` через IPv6, но сервер слушает только на IPv4.

---

## ✅ Решение (30 секунд)

### Шаг 1: Скачайте проект
```bash
git clone <repository-url>
cd NoviVovi
```

### Шаг 2: Запустите скрипт исправления

**Правой кнопкой мыши** на файл `fix-localhost-production.ps1` → **"Запуск с PowerShell"**

Или через командную строку:
```powershell
powershell -ExecutionPolicy Bypass -File fix-localhost-production.ps1
```

### Шаг 3: Подтвердите UAC

Когда появится запрос прав администратора - нажмите **"Да"**

### Шаг 4: Готово!

Скрипт автоматически:
- ✅ Создаст резервную копию файла hosts
- ✅ Исправит конфигурацию localhost
- ✅ Проверит, что всё работает

---

## 🛡️ Безопасность

**Скрипт полностью безопасен:**
- Создаёт резервную копию перед любыми изменениями
- Не удаляет существующие записи
- Только добавляет/раскомментирует localhost записи
- Все бэкапы сохраняются в `C:\Windows\System32\drivers\etc\backups\`

---

## 🔍 Что делает скрипт?

Скрипт проверяет файл `C:\Windows\System32\drivers\etc\hosts` и добавляет/раскомментирует строки:

```
127.0.0.1       localhost
::1             localhost
```

Это стандартная конфигурация Windows, которая иногда бывает закомментирована.

---

## ❓ Если что-то пошло не так

### Вариант 1: Восстановить из бэкапа

Скрипт создаёт бэкапы в папке:
```
C:\Windows\System32\drivers\etc\backups\
```

Чтобы восстановить, запустите PowerShell **от имени администратора**:
```powershell
Copy-Item "C:\Windows\System32\drivers\etc\backups\hosts.backup.YYYYMMDD-HHMMSS" "C:\Windows\System32\drivers\etc\hosts" -Force
```

### Вариант 2: Исправить вручную

1. Откройте **Блокнот от имени администратора**:
   - `Win + S` → введите "Блокнот"
   - Правой кнопкой → "Запуск от имени администратора"

2. Откройте файл:
   - Файл → Открыть
   - Вставьте путь: `C:\Windows\System32\drivers\etc\hosts`
   - Измените фильтр на "Все файлы (*.*)"

3. Найдите строки:
   ```
   #	127.0.0.1       localhost
   #	::1             localhost
   ```

4. Раскомментируйте (уберите `#` и табуляцию):
   ```
   127.0.0.1       localhost
   ::1             localhost
   ```

5. Сохраните файл (Ctrl+S)

## 🎯 Проверка

После исправления проверьте:

```bash
ping localhost
```

Должен отвечать либо `127.0.0.1`, либо `::1` - оба варианта правильные.

Затем запустите API:
```bash
cd backend/NoviVovi.Api
dotnet run
```

И протестируйте:
```bash
curl http://localhost:5136/api/novels
```

## 💡 Для разработчиков на Java/Kotlin

Если используете Java/Kotlin клиенты, добавьте JVM параметр:
```
-Djava.net.preferIPv4Stack=true
```

**IntelliJ IDEA:**
- Run → Edit Configurations → VM options

**Gradle:**
```gradle
test {
    jvmArgs '-Djava.net.preferIPv4Stack=true'
}
```
